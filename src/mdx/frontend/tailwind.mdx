# TailwindCSS

## Installation

```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

## Basic Setup

```javascript
// tailwind.config.js
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    './src/**/*.{js,ts,jsx,tsx,mdx}',
  ],
  theme: {
    extend: {
      colors: {
        primary: '#3B82F6',
        secondary: '#10B981',
      },
    },
  },
  plugins: [],
}
```

```css
/* globals.css */
@tailwind base;
@tailwind components;
@tailwind utilities;
```

## Tips and Best Practices

### Organization
- Use @layer components for reusable styles
- Keep custom utilities in theme.extend
- Use meaningful color names

### Performance
- Purge unused styles in production
- Use JIT mode for faster development
- Minimize custom CSS

### Maintainability
- Use consistent naming conventions
- Document custom utilities
- Keep configuration organized

## Common Patterns

### Responsive Design
```jsx
<div className="
  w-full 
  md:w-1/2 
  lg:w-1/3
  p-4
  hover:shadow-lg
  transition-all
">
  Responsive Container
</div>
```

### Custom Components
```jsx
<button className="
  px-4 
  py-2 
  bg-primary 
  text-white 
  rounded-lg
  hover:bg-primary/90
  transition-colors
">
  Custom Button
</button>
```

### Dark Mode
```jsx
<div className="
  bg-white 
  dark:bg-gray-800
  text-gray-900
  dark:text-gray-100
">
  Dark Mode Compatible
</div>
```

### Grid Layout
```jsx
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <div className="p-4 bg-gray-100">Item 1</div>
  <div className="p-4 bg-gray-100">Item 2</div>
  <div className="p-4 bg-gray-100">Item 3</div>
</div>
```
